4단계

인자 없는 /unguard가 설정 과정을 대화형으로 안내하도록 하여, 우회를 설정할 때 더 이상 인자 문법을 외울 필요가 없게 했습니다.

Unguard Interactive

왜

인자 없는 /unguard는 상태만 출력했고, 설정하려면 여전히 인자 문법(write 30m, all persist, …)을 외워야 했습니다. Doctor Cho는 인자 없는 호출이 설정 과정을 대화형으로 안내하기를 원했습니다.

무엇을

인자 없는 /unguard는 이제: 현재 상태를 보여준 뒤, 두 개의 질문으로 하나의 AskUserQuestion을 던집니다 — Action(write / read / all / off, 이미 ON인 옵션은 표시됨)과 Scope(이번 세션만 / 30m / 2h / persist, "Other"는 45m 같은 사용자 지정 지속 시간을 그대로 전달) — 그리고 선택 결과로 토글을 실행합니다. 인자가 있으면 동작은 그대로입니다(전달만). /unguard status는 질문 없이 그냥 확인하는 방법으로 유지됩니다.

어떻게

커맨드 markdown만 바꾼 변경입니다: ! 줄이 이미 전달된 인자를 렌더링하므로, 지시문은 인용된 문자열이 비어 있는지 여부로 분기합니다. allowed-tools에 AskUserQuestion이 추가되었습니다. 모델은 스스로 action/scope를 고르지 않도록 지시받으며 — disable-model-invocation은 여전히 모델이 명령을 아예 호출하지 못하도록 막습니다. 스크립트 변경은 없으며, 두 테스트 스위트 모두 그대로 통과합니다(68 + 26).
