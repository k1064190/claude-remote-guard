3단계

기존에 있는 어떤 상태 표시줄 위에도 겹쳐 구성되는 상태 표시줄 인디케이터를 추가하여, 사용자가 자신의 스크립트를 직접 편집하지 않고도 가드의 armed/bypassed 상태를 볼 수 있게 했습니다.

Statusline Indicator

왜

사용자들은 가드 상태(armed 또는 bypassed)를 상태 표시줄에서 보고 싶어 했지만, Claude Code는 단 하나의 statusLine.command만 렌더링하며 플러그인이 여기에 등록하거나 쌓거나 덧붙일 방법을 제공하지 않습니다. 예전 README는 사용자가 자신의 상태 표시줄 스크립트를 직접 편집하라고 안내했는데, 이는 취약했고 상태 표시줄 전체를 소유하는 다른 플러그인을 사용하는 사람에게는 아예 불가능했습니다.

무엇을

/guard-statusline [install|uninstall|status]:

  • install은 이미 있는 상태 표시줄 위에 가드 줄을 겹쳐 구성하고 statusLine이 래퍼를 가리키도록 재설정함(멱등적; 갱신하려면 재실행)
  • uninstall은 원래의 statusLine 객체를 그대로 복원
  • status는 현재 연결 상태와 가드 상태를 보고
  • "🔒 guard: armed" / "🔓 guard: R|W|RW bypass"를 마지막 줄에 자체적으로 렌더링

어떻게

스크립트 두 개로 구성됩니다. statusline-guard.sh는 렌더링 시점 래퍼로, 기록된 내부 명령을 stdin을 그대로 유지한 채(eval, 선택적으로 설정되지 않은 변수 때문에 중단되지 않도록 set +u 서브셸에서) 실행하고, 그 출력을 그대로 찍은 뒤, bypass-read|write 마커 파일에서 읽은 가드 줄을 덧붙입니다(플래그 내용/만료를 존중하며 읽기 전용). guard-statusline-setup.sh는 jq를 사용해 한 번만 수행되는 연결 작업을 합니다: 현재 statusLine.command(문자열)와 전체 .statusLine 객체를 저장하고(정확한 복원을 위해), 기존 statusLine 위에 병합하며(padding 같은 형제 필드를 보존), 래퍼를 안정적인 경로 (~/.claude/remote-guard/statusline.sh)로 복사해 플러그인 버전이 올라가도 settings 항목이 유효하게 유지되도록 하고, settings를 원자적으로 씁니다(temp+rename, 심볼릭 링크된 dotfile을 이식성 있게 처리). 명령 인자는 첫 토큰 확장으로 파싱합니다(글롭 없음). require_valid_settings는 손상된 JSON을 거부하며, warn_if_shadowed는 가장 가까운 상위 .claude/까지 거슬러 올라가 프로젝트/로컬 statusLine이 사용자 수준의 설정을 가릴 때 경고합니다. 플러그인 설치 시 자동 설치는 불가능하므로(지원되는 훅으로는 settings.json을 편집할 수 없음), 이 명령이 사용자가 취해야 할 최소한의 행동이며 disable-model-invocation으로 설정되어 있습니다.

리뷰 루프

PR 이전에 code-reviewer-pro, Codex (gpt-5.5), Antigravity (Gemini 3.1 Pro)가 리뷰했습니다: 공백이 있는 경로의 따옴표 처리, jq --arg/--argjson (인젝션 방지), 같은 파일시스템 내 원자적 settings 쓰기, 손상된 JSON 거부, stdin 개행 보존, 원자적 래퍼 복사, 심볼릭 링크된 settings에 대한 write-through, 인자 글롭 안전성을 수정했습니다.

이후 Codex GitHub 봇이 4라운드를 진행했습니다: 1차 — 만료된 플래그 표시, 백업 권한 0600/0700, 프로젝트-shadow 경고, 형제 statusLine 키 보존; 2차 — 실수로 커밋된 스크래치 파일 제거, printf %q 경로 이스케이핑, 원자적 심볼릭 링크 쓰기, README 수동 스니펫 만료; 3차 — macOS/BSD를 위한 이식성 있는 resolve_symlink, 내부 eval 주변의 set +u, 상위 경로 순회 shadow 검사, 플러그인 제거 전 uninstall 안내 문서; 4차 — 이상 없음("Didn't find any major issues"). 4라운드 동안 총 20건이 발견되어 20건 수정, 0건 보류되었습니다. 테스트가 22개 → 43개로 늘었습니다.

코드 위치

  • scripts/statusline-guard.sh — 렌더링 시점 합성 래퍼(stdin passthrough + 가드 줄)
  • scripts/guard-statusline-setup.sh — install/uninstall/status; jq settings 편집, 안정 경로 복사, 멱등성, 정확한 복원
  • commands/guard-statusline.md — 슬래시 명령(Bash, disable-model-invocation)
  • tests/test-guard-statusline.sh — assert 43건
  • README.md — "Optional: status-line indicator" 섹션

회고

합성 방식(내부 명령 기록 → 감싸기 → 복원)이 어떤 상태 표시줄과도 충돌 없이 동작하게 해주는 핵심입니다. 안정 경로로 복사하는 단계 덕분에 플러그인이 업데이트되어도 settings.json이 유효하게 유지됩니다. 각 리뷰 라운드가 실제 문제를 잡아냈고, 이전 수정의 회귀까지 포함해서 — 적대적 다중 리뷰어 패스는 제값을 했습니다.
